# Clippy

A lightweight, native macOS clipboard manager. The popup opens at the text
cursor, where you actually need it, instead of a fixed bar at the bottom of
the screen.

Built per `clipboard-manager-stack-decision.md`: Swift, SwiftUI inside a
nonactivating `NSPanel`, Accessibility API for caret positioning, GRDB/SQLite
with FTS5 search. Local-only; nothing leaves the machine.

## Build and run

Requires macOS 14+, Swift 6 (Xcode 16+ or current Command Line Tools).

```sh
./scripts/make-app.sh        # builds release and assembles build/Clippy.app
open build/Clippy.app
```

For development, `swift build && .build/debug/Clippy` works too, but use the
.app bundle for daily use: the bundle id keeps the Accessibility grant stable
across rebuilds.

On first launch macOS asks for Accessibility permission. Grant it in
System Settings > Privacy & Security > Accessibility. It is needed for two
things: reading the caret position to place the panel, and sending the
Cmd-V keystroke that pastes the selected clip. Without it the panel opens at
the mouse pointer and selecting a clip copies it instead of pasting.

## Use

- Cmd+Shift+V opens the panel at the text cursor (falls back to the mouse
  pointer in apps that hide the caret, e.g. Electron apps).
- Clips render as cards: source app icon, per-app color stripe and tint,
  content-type badge (link, email, color, file path, text), and a live
  swatch for copied color values. Date headers group the list (Pinned,
  Today, Yesterday, This Week, This Month, Earlier).
- Tabs: History (Cmd+1) and Pinned (Cmd+2). The Pinned tab is the
  management pane for everything you keep.
- Type to search (FTS5, prefix matching). Arrow keys move, Return pastes.
- Shift+Return pastes in the non-default mode (plain vs formatted).
- Cmd+E edits the selected clip in a plain-text editor with every smart
  substitution disabled; straight quotes and hyphens survive untouched.
- Cmd+P pins a clip (pinned clips float to the top and survive history caps
  and Clear). Cmd+Delete deletes. Esc or clicking elsewhere closes.
- Hover a card for quick actions (paste plain, edit, pin, delete);
  right-click for the full menu.
- Menu bar icon: open panel, pause capture, settings, clear unpinned
  history, quit.

## Settings

Four tabs:

- General: paste defaults, move-to-top, history cap, launch at login.
- Appearance: light/dark/system, 10 accent colors, panel material (glass
  to solid), card color mode (by source app like Paste, by content type,
  accent, or neutral), app icons on cards, date headers, popup position
  (caret / mouse / last position / center), panel size.
- Capture: polling interval (100-1000 ms), per-app ignore list.
- Integrations: export history as JSON, reveal the database in Finder,
  plus the planned MCP server, REST API, sync, and encryption surface.

## Debug flags

- `--show-panel` opens the popup right after launch.
- `--screenshot <path>` renders the popup to a PNG and exits (used for UI
  smoke tests).

## Privacy and capture rules

- Items marked `org.nspasteboard.ConcealedType` (password managers like
  1Password/Bitwarden), `TransientType`, or `AutoGeneratedType` are never
  stored.
- Apps on the ignore list are never captured.
- Data lives in `~/Library/Application Support/Clippy/clippy.sqlite`,
  local-only. Encryption at rest (SQLCipher) is the next milestone.

## Layout

```
Sources/Clippy/
  main.swift                     app entry, accessory activation policy
  AppDelegate.swift              wiring: status item, hotkey, services
  Capture/ClipboardMonitor.swift NSPasteboard changeCount polling + filters
  Storage/Clip.swift             record model
  Storage/ClipDatabase.swift     GRDB queue, migrations, FTS5, queries
  Positioning/CaretLocator.swift AX caret rect + coordinate conversion
  Panel/PastePanel.swift         nonactivating borderless key-capable panel
  Panel/PanelController.swift    placement modes, clamping, show/hide
  Panel/EditorWindowController.swift
  Paste/PasteService.swift       pasteboard write + simulated Cmd-V
  Support/HotKeyCenter.swift     Carbon global hotkey
  Support/AppSettings.swift      all user-facing knobs, UserDefaults-backed
  UI/                            SwiftUI views (list, editor, settings)
```

## Roadmap (from the stack decision doc)

1. Pinboards (named collections; schema already supports the pinned flag)
2. SQLCipher encryption at rest, key in the Keychain
3. MCP server (official Swift SDK) + loopback REST API
4. User-controlled sync (synced folder / CloudKit / encrypted export)
5. Custom hotkey recording
